Qt的布局系统很强大，官方文档写得很清楚：用了布局，窗口缩放自适应，跨平台兼容，各种好处。但没人告诉我的是：**当控件数量超过一百个的时候，布局微调就是一场噩梦。**

## 那个让我崩溃的项目

当时在做一个信息查询系统，对接多个不同平台的数据。界面用的是 `QStackedWidget`，分了十多个页面，每个页面七八十个控件，加起来一百多个。

最崩溃的是调 `QLineEdit` 的长度。有的显示内容多，需要拉长；有的就几个字，短一点就行。结果是什么呢？

**一个个手动拉好长度 → 一用布局，全乱了 → 取消布局 → 重新拉 → 再试布局 → 又乱了 → 再取消……**

循环往复，一下午就过去了。

## 血的教训总结出来的技巧

踩了无数次坑之后，总算总结出点东西：

### 1. 复制粘贴大法

不要一个一个拖控件。做好第一行，**整行复制**，然后改名称。这样能保证整体对齐，布局跑起来不至于太离谱。

```cpp
// 伪代码示意
for (int i = 0; i < 10; ++i) {
    QLineEdit *edit = new QLineEdit(this);
    edit->setObjectName(QString("lineEdit_%1").arg(i));
    gridLayout->addWidget(edit, i, 1);
}
```

### 2. Spacing 设为 0

Qt 默认每层布局的 spacing 是 9，看起来松松垮垮：

```cpp
// 紧凑一点
gridLayout->setSpacing(0);
gridLayout->setContentsMargins(5, 5, 5, 5);
```

### 3. 该放弃时就放弃

如果界面实在太复杂，别死磕布局。某些固定大小的对话框，手动设坐标反而更省事。布局是为了**大部分情况**省力，不是为了**所有情况**都要用。

## 现在回头看

如果现在让我再做这样一个界面：
- 能用代码生成控件，就别手动拖
- 复杂界面考虑分拆成多个子窗口
- 先确定设计稿，再动手，不要边做边调

**但最想说的是：放过自己吧，这种界面做一次就够了。** 🙏
