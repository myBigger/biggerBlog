# 必哥手记 · 写作规则

> 本规则定义了「必哥手记」专栏文章的写作标准。
> 适用场景：撰写新文章、修改已有文章、重写草稿。

---

## 一、语言与语气

**语言**：简体中文，混合技术英文术语（Qt、C++、API 等保留英文，不翻译）
**语气**：第一人称（「我」「必哥」），真实、直接、有态度
**受众**：有实际开发经验的中文工程师，不需要从零解释概念

### 语气示例

❌ 不好（太泛、太正确）：
> "在使用 Qt 进行网络编程时，SSL 配置是一个重要的环节。本文将介绍..."

✅ 好（直接、真实）：
> "Qt5 跑得好好的，升级 Qt6 之后，SSL 请求全部报错。一行代码没动，库一换，整套认证流程全断了。"
> "当时我以为是指令集不兼容，查了两天，最后发现是 Windows Server 2016 的根证书库少了一个中间 CA。"

## 二、标题规范

**格式**：`「必哥手记」| 第 N 期 · {核心主题}`

如果文章不在必哥手记专栏，也用描述性标题，但**永远不要**用：
- "XXX 详解"（太泛）
- "深入理解 XXX"（太标题党）
- "关于 XXX 的一些思考"（太水）

✅ 好的标题：
- `Qt 网络编程的真相：为什么你的 QSslSocket 总是在关键时刻掉链子`
- `CAS 单点登录协议实战：TGT/ST 交换的每一个细节`
- `Fedora ARM64 编译 Windows Qt 程序：血泪经验总结`

---

## 三、字数标准

| 类型 | 参考字数 |
|------|---------|
| 必哥手记（深度长文） | 3000-6000 字 |
| 踩坑记录 | 1500-3000 字 |
| 工具介绍 | 800-1500 字 |
| 快讯/短评 | 500-800 字 |

质量优先，字数是参考，不是目标。宁可 2000 字讲透一个问题，也不要 5000 字说一堆正确的废话。

---

## 四、代码块规范

**语言标注**：所有代码块必须标注语言（cpp、bash、yaml、json、typescript 等）

**格式要求**：
- 关键行加注释（`//` 或 `#`），解释「为什么这样写」，不只写「做什么」
- 错误代码和正确代码对比，用 `// ❌` / `// ✅` 标注

**示例**：

```cpp
// ❌ 错误：QQueue 是 incomplete type，必须 include
QQueue<QString> queue;

// ✅ 正确：使用 QList，或显式 include <QtCore/QQueue>
// #include <QtCore/QQueue>  // 不推荐隐式依赖
QList<QString> queue;
```

## 五、结构要求

### 引子（必须）

前 200 字内必须出现：
- 一个**具体场景**或**真实痛点**
- 让读者能快速判断「这说的是我的问题」

不接受的引子写法：
- 上来就下定义（"CAS 是..."）
- 上来就列提纲（"本文将从以下几个方面..."）
- 上来就讲历史（"早在2001年，Yale University 设计了..."）

### 正文

- 每个章节有**一句核心观点**作为开头，后面跟解释和证据
- 避免「首先、其次、最后」的教条结构——用自然的段落流动
- 结论先行，论据跟上

### 结语

- 一句话总结核心教训（不超过 50 字）
- 可以加一句延伸思考或下期预告

---

## 六、SEO 和元信息

**description 字段**：60-160 字，必须是完整的句子，能独立传达文章价值。不得用"本文介绍..."开头。

✅ 好的 description：
> Qt 网络编程的真相：QSslSocket 在 Qt5/Qt6 切换时的 SSL 行为差异，Windows/macOS/Linux 三大平台根证书库的组织方式，以及最容易踩的 5 个 SSL 配置坑。

❌ 坏的 description：
> 本文介绍了 Qt 网络编程中 SSL 相关的配置问题。

---

## 七、标签使用

文章必须有 tags，且遵循以下规则：
- 每篇文章至少 2 个 tag，最多 5 个
- `必哥手记` 只在专栏文章中使用
- 原创技术内容用真实技术 tag，不滥用 `原创`

常用 tag 列表：
```
Qt, C++, 架构设计, 开源, 踩坑, 生产事故,
必哥手记, 工具链, 职业成长, 网络编程,
SSL/TLS, CAS, GitHub Actions, Vercel
```

---

## 八、修订已有文章

修改已有文章时：
- 不改原文的核心观点和故事线
- 可以修正错误、补充新的实践经验
- 如果发现文章观点需要大改，**先与必哥沟通**，不要自行大改
- commit message 写清楚改了什么（"修正第3期 SSL 配置的表述错误"）
